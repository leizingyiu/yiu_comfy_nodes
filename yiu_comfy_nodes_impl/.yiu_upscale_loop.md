# yiu_upscale_loop（START / END）

## 1. 目标与定位

`yiu_upscale_loop` 用于把“放大/超分”这类操作做成一个可循环的流程：  
用户只需要把自己的放大节点（或节点组）夹在 `START` 与 `END` 之间，就能把一次性放大改成“分多次放大”，直到达到目标尺寸/倍率。

它以“前后两个节点”的形式提供：

- `yiu_upscale_loop_START`：负责计算本轮应该放大的倍率，并把本轮要处理的 tiles 批量与倍率输出给用户的放大节点（tiling 在内部固定执行；想关闭 tiling 则把 `max_tile_size` 设为 0）。
- `yiu_upscale_loop_END`：接收用户放大后的图像，更新循环状态，决定是否继续下一轮，并在循环结束时输出最终结果。

两者之间有一根类似 `forLoop` 的专用连线 `flow`（FLOW_CONTROL），用于驱动循环与携带状态。

## 2. 节点命名

- `yiu_upscale_loop_START`
- `yiu_upscale_loop_END`

## 3. 连接方式（最重要）

最小连接如下（不含切片）：

1. `START.image` → 你的放大节点的 `image` 输入（START 会在内部先 tile；所以你收到的是 tiles 批量）
2. `START.scale_this_time` → 你的放大节点的 `scale` / `upscale_by`（如果你的放大节点需要倍率）
3. `START.loop_state` → `END.loop_state`
4. `START.flow` → `END.flow`
5. 你的放大节点 `image` 输出 → `END.image_upscaled`（这里仍然是 tiles 批量）

可选（如果你的放大节点不吃倍率）：

- 你可以忽略 `START.scale_this_time`，用固定倍率的放大节点；但此时循环收敛到目标的行为将取决于你的固定倍率设置。

## 4. 解决什么问题

典型场景：

- 想把一张图放到目标宽高，但一次性放大倍率太大，容易糊、容易出伪影，或显存/内存不够。
- 想把总倍率拆成多次（例如每次 1.5x 或 2.0x），最后一步自动用一个“补足倍率”精确到目标。
- 想在每次放大之间插入任意自定义的放大器（模型超分、SD 放大、含锐化/降噪/重采样等的节点组）。

## 5. 工作原理（与参考工作流一致的核心逻辑）

整体目标可以理解为：把“总倍率（total_scale）”拆成多步执行。

- 由输入图像的 `(w, h)` 与目标 `(target_w, target_h)` 计算总倍率：
  - `total_scale = max(target_w / w, target_h / h)`
- 用 `scale_every_time` 作为每一步的基础倍率，计算需要多少步：
  - `scale_total_times = ceil( log(total_scale, scale_every_time) )`
- 每次循环输出当前步倍率 `scale_this_time`：
  - 大多数步：`scale_this_time = scale_every_time`
  - 最后一步：自动给出“补足倍率”，让累计倍率精确到 `total_scale`（避免最后结果偏大/偏小）

循环结束时，`END` 输出最终放大后的图像（以及可选的调试信息/统计信息）。

## 6. 接口设计（产品层规格）

### 6.1 yiu_upscale_loop_START

**输入（绿色框：前节点参数）**

- `image`（IMAGE）：要放大的输入图像。
- `target_width`（INT）：目标宽度（像素）。
- `target_height`（INT）：目标高度（像素）。
- `scale_every_time`（FLOAT）：每一步的基础放大倍率（建议 1.2 ~ 2.0）。
- `max_tile_size`（INT，可选）：tile 最大边长（像素）。建议 1024/1536。设为 0 表示关闭 tiling（整图直通）。
- `min_tile_overlap_px`（INT，可选）：最小接缝宽度（像素）。对应你工作流里的 `default_OVERLAP` 这种“具体宽度”设定。设为 0 表示最小值为 0。
- `tile_overlap_ratio`（FLOAT，可选）：切片重叠比例（0~1）。当图变大时，让接缝宽度随尺寸同比例增大（例如 0.1~0.25）。

说明：

- `target_width/target_height` 可以只设一个维度：另一维可由节点内部按原图比例推导（具体实现以最终节点为准）。产品层建议同时填写以避免歧义。
- 若 `target_width <= w` 且 `target_height <= h`，则可视为不需要放大；节点应给出防呆提示，并直接走“单步/零步”输出逻辑。
- 重叠宽度推荐采用“比例 + 最小像素值”的混合策略，兼顾小图与大图：  
  `overlap_px = max(min_tile_overlap_px, round(tile_size * tile_overlap_ratio))`  
  这样小图时至少有 `min_tile_overlap_px` 的接缝宽度，大图时接缝会按比例变宽以更平滑地过渡。

**输出**

- `flow`（FLOW_CONTROL）：循环连线，连接到 `END.flow`。
- `image`（IMAGE）：本轮要交给用户放大节点处理的 tiles 批量（循环内会自动更新为上一步输出）。
- `scale_this_time`（FLOAT）：本轮推荐的放大倍率，供中间的放大节点使用。
- `loop_state`（YIU_UPSCALE_LOOP_STATE）：内部循环状态，连接到 `END.loop_state`。
- `print(step_index)`（INT，可选）：当前第几步（从 0 开始）。
- `print(step_total)`（INT，可选）：总步数。
- `help_text`（STRING，可选）：节点说明/调试信息（遵循 YiuBaseNode 的 help_text 习惯）。

### 6.2 yiu_upscale_loop_END

**输入**

- `flow`（FLOW_CONTROL）：来自 `START.flow` 的循环连线。
- `loop_state`（YIU_UPSCALE_LOOP_STATE）：来自 `START.loop_state` 的内部循环状态。
- `image_upscaled`（IMAGE）：来自用户放大节点（或节点组）的放大结果（会回传作为下一步输入图像）。

**输出（紫色框：后节点输出）**

- `image`（IMAGE）：循环完成后的最终图像（END 内部会把 tiles 拼回整图）。
- `image_step`（IMAGE，可选）：每一步（每次循环）产出的整图结果（更适合在循环体内外接保存节点做“防崩溃容错保存”）。
- `print(last_scale_applied)`（FLOAT，可选）：本步实际使用的倍率（来自 START 的 `scale_this_time`）。
- `print(step_index)`（INT，可选）：当前第几步（便于用于文件名/日志）。
- `print(step_total)`（INT，可选）：总步数。
- `help_text`（STRING，可选）：节点说明/调试信息。

## 7. 使用建议

- `scale_every_time` 不要设置过大；更推荐 1.3~1.8 这种“多次、小倍率”方式，最后一步由节点自动补齐。
- 中间放大节点尽量做到“以倍率为输入、以图像为输出”，并避免在循环内引入随机性（否则每一步风格可能漂移）。
- 需要拼接时优先启用切片，并设置合理重叠；重叠越大越不容易出接缝，但越吃算力与内存。
- 需要“每一步自动保存”时，不建议在节点内部直接写保存逻辑；更推荐使用 `END.image_step` 外接任意保存节点（例如 SaveImage / 你自己的 yiu 保存节点），这样保存策略（路径、命名、是否启用）完全由用户工作流控制。

## 8. 兼容性与约束

- 本功能设计为“两个节点夹一个用户自定义放大模块”的形态，要求用户放大模块至少能输出 `IMAGE`。
- 循环连线 `flow` 是一条专用连接：必须连接 `START.flow → END.flow`，否则循环不会工作或无法收敛。
- 节点实现将基于 `YiuBaseNode` 的输入输出定义与 `help_text` 约定。

## 9. 最小示例（概念图）

```
image
  │
  ▼
yiu_upscale_loop_START ───── loop ─────► yiu_upscale_loop_END
  │  image                         ▲
  ├────────► 你的放大节点/节点组 ───┤ image_upscaled
  │  scale_this_time (可选)
  └────────► (接到放大倍率输入)
 
注：此处的 loop 连线在实现中对应 `flow`（FLOW_CONTROL）。
```
